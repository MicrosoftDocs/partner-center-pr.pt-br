## <a name="changes-coming-to-partner-center-apis"></a><a name="9"></a>Próximas mudanças nas APIs do Partner Center

### <a name="categories"></a>Categorias

- Data: 28/07/2020
- Capacidades

## <a name="summary"></a>Resumo

A equipe do Partner Center avaliou as APIs do Partner Center expostas. Vemos que algumas dessas APIs estão obsoletas e não têm mais suporte. Elas serão desativadas até 31 de agosto de 2020.

## <a name="details"></a>Detalhes

A equipe do CSP Partner Center identificou algumas APIs que estamos agendando para serem desativadas até 31 de agosto de 2020. Essas APIs representam interfaces que foram criadas há anos, muitas das quais não estão mais funcionais. A equipe está no processo de remover referências a esses pontos de extremidade na próxima versão do SDK, no último trimestre calendário de 2020. Abaixo estão as APIs a serem encerradas. Os parceiros não devem referenciar esses URIs de agora em diante.

**Solicitações de serviço de parceiro** – esses pontos de extremidade permitiam que um parceiro obtivesse cadeias de caracteres de tópico de suporte e enviasse solicitações de serviço de parceiro programaticamente. Ao longo do tempo, essa API tornou-se instável e não tem mais suporte. O parceiro é direcionado para a interface do usuário do Partner Center para criar tíquetes de suporte do parceiro. A experiência do usuário oferece fornece mais informações aos parceiros durante a criação de casos de suporte, como etapas recomendadas e documentos para a área em que estão tendo problemas. A equipe do Partner Center também aprimorou a experiência de criação de tíquete de parceiro, personalizando os formulários de solicitação de serviço para capturar informações específicas da área problemática. As informações coletadas pelos formulários online ajudam os engenheiros de suporte a resolver os problemas deles com mais rapidez e precisão. Esses URIs não devem ser referenciados nem usados e serão encerrados até o fim de agosto de 2020. Perceba que essas mudanças afetam apenas os tópicos de suporte e criam métodos de solicitação de serviço de **parceiro**; todas as outras APIs relacionadas ao suporte devem continuar funcionando conforme o esperado.

- {base-URL}/v1/servicerequests/supporttopics
- {base-URL}/v1/servicerequests/{agent-locale}

**Incidentes de serviço do Office** – esses pontos de extremidade foram criados para permitir que um parceiro obtivesse uma exibição agregada de incidentes/interrupções de serviço em todos os serviços dos parceiros deles. No entanto, os dados retornados por esses pontos de extremidade se tornaram obsoletos e não são mais úteis. Vamos encerrar esses pontos de extremidade no final de julho. Prosseguindo, a equipe do Partner Center reavaliará o incidente do serviço de "todos os clientes" e os cenários de integridade e procurará uma oportunidade de agregar valor nessa área. As referências a esses URIs devem ser removidas.

- {base-URL}/v1/serviceincidents/office/count
- {base-URL}/v1/serviceincidents/office

Os parceiros podem usar a [API de Comunicações de Serviço do Office 365](https://docs.microsoft.com/office/office-365-management-api/office-365-service-communications-api-reference) para obter o status atualizado dos serviços do cliente.

**TopProduct** – esse ponto de extremidade foi criado como uma prova de conceito que nunca foi destinada a disponibilidade geral. Ele foi criado para, algum dia, permitir que os parceiros gerenciassem as principais ofertas que seriam renderizadas no catálogo do Partner Center. Essa API, no entanto, nunca foi totalmente desenvolvida e concluída e será desabilitada até 31 de julho de 2020. Os parceiros devem remover referências a ela.

- {base-URL}/v1/preferences/catalog/products

## <a name="next-steps"></a>Próximas etapas

- Os parceiros devem verificar se as soluções deles não fazem mais referência às APIs acima.
- Os parceiros do Provedor de Soluções na Nuvem poderão acessar a [Comunidade do Yammer de SDK e API do Partner Center](https://www.yammer.com/cloudpartnercommunity/#/Threads/show?threadId=786951767490560) para fazer perguntas se forem necessários mais esclarecimentos.

## <a name="sdk-release-on-net-standard-v1161-preview"></a><a name="8"></a>Versão do SDK no .NET Standard (v1.16.1-preview)

### <a name="categories"></a>Categorias

- Data: 21/07/2020
- Capacidades

### <a name="impacted-audience"></a>Público-alvo afetado

Parceiros do programa CSP Direct Bill Partner e fornecedores do programa CSP Indirect Provider que estão usando o SDK do .NET do Partner Center. 

### <a name="summary"></a>Resumo

Desde 21 de julho de 2020, os parceiros podem baixar a versão prévia do SDK do .NET v1.16.1-preview **SDK do .NET Standard do Microsoft Partner Center**, juntamente com os **exemplos do GitHub do SDK do Partner Center** públicos atualizados.

### <a name="details"></a>Detalhes

Parceiros do programa CSP Direct Bill Partner e fornecedores do programa CSP Indirect Provider são qualificados para baixar a versão prévia mais recente do SDK do .NET Standard. Vamos migrar o SDK do Microsoft Partner Center do .NET Framework para a plataforma .NET Standard 2.0. Isso tornará o SDK compatível com o aplicativo existente usando o .NET Framework 4.6.1 e versões mais recentes. O .NET Standard dará suporte ao .NET Core 2.0 e versões mais recentes.

Houve uma enorme demanda de parceiros solicitando o SDK no .NET Standard. A versão atual do SDK força os parceiros a usar aplicativos escritos em ASP.NET Framework, que é lento e caro em comparação com aplicativos .NET Core. Portanto, a migração para o .NET Standard pode ajudar a resolver os problemas com o desempenho ou a compatibilidade com o .NET Core.
 
**Destaques**
- Não há mais versões da Microsoft para .NET Framework 4.8, pois ela é a última da linha.
- Limita o desenvolvimento futuro, pois a maioria das bibliotecas será lançada para .NET Core ou .NET Standard, que pode não dar suporte à versão atual da estrutura.
- O .NET Core é favorável ao uso com contêineres devido à natureza leve dele.
- O runtime do servidor de alto desempenho para Windows Server e Linux torna o .NET Core uma estrutura da Web de excelente desempenho.

### <a name="next-steps"></a>Próximas etapas

- Baixar a última versão do [SDK do .NET v1.16.1-preview](https://www.nuget.org/packages/Microsoft.Store.PartnerCenter/1.16.1-preview)
- Versão: SDK do .NET v1.16.1-preview 
- Código de exemplo: [Código de exemplo](https://github.com/microsoft/Partner-Center-DotNet-Samples)
- Versão prévia disponível em: 21/07/2020 
- GA: três semanas a contar da visualização (para dar tempo suficiente para os parceiros migrarem) 
- Não há novas alterações da API como parte da versão atual

## <a name="cloud-solution-provider-csp-incentive-for-eligible-microsoft-commercial-marketplace-offers-enhanced-for-fy21"></a><a name="7"></a>Aprimoramento do incentivo do CSP (Provedor de Soluções na Nuvem) para ofertas do marketplace comercial da Microsoft qualificadas para o ano fiscal de 2021

### <a name="categories"></a>Categorias

- Data: 22/07/2020
- Capacidades

### <a name="impacted-audience"></a>Público-alvo afetado

Parceiros do programa CSP Direct Bill Partner e fornecedores do programa CSP Indirect Provider

### <a name="summary"></a>Resumo

Desde 1º de julho de 2020, parceiros do programa CSP Direct Bill Partner e fornecedores do programa CSP Indirect Provider podem obter incentivos em transações de ofertas de IaaS (infraestrutura como serviço) qualificadas.

### <a name="details"></a>Detalhes

Os parceiros do programa CSP Direct Bill Partner e os fornecedores do programa CSP Indirect Provider são qualificados para realizar transações de ofertas de terceiros selecionadas para os clientes deles no [marketplace comercial da Microsoft](https://azure.microsoft.com/overview/commercial-marketplace). Como recompensa por aumentar a receita do marketplace comercial, os parceiros do programa CSP Direct Bill Partner e os fornecedores do programa CSP Indirect Provider qualificados ganham um incentivo sobre transações de ofertas qualificadas.

Desde 1º de julho de 2020, você pode ganhar um incentivo sobre transações de ofertas de IaaS qualificadas. O incentivo para ofertas de SaaS (software como serviço) qualificadas continuará no ano fiscal de 2021.

Você pode encontrar a lista mais recente das ofertas de SaaS e IaaS qualificadas por incentivo no [site de Parceiro da Microsoft](https://aka.ms/partnerincentives). A lista de ofertas qualificadas é atualizada mensalmente e postada nas coleções de ativos para os programas de incentivo do CSP Direct Bill Partner e do CSP Indirect Provider. Os parceiros CSP podem iniciar essas transações por meio do Partner Center ou do portal do Azure.

Para ganhar incentivos, você precisará atender a todos os requisitos de qualificação e registro, conforme descrito no respectivo guia do programa de incentivos. Os guias do programa e outros recursos úteis de incentivos estão disponíveis no [site do parceiro](https://aka.ms/partnerincentives).

### <a name="next-steps"></a>Próximas etapas

- Confira o comunicado completo e saiba mais sobre o marketplace comercial na coleção da galeria de recursos de Preparação para Operações: [Incentivo do CSP para ofertas qualificadas do marketplace comercial da Microsoft](https://partner.microsoft.com/resources/collection/csp-incentive-for-microsoft-commercial-marketplace-offers#/).
 
- Examine a lista de ofertas de SaaS e IaaS qualificadas no início de cada mês.
 
- Envolva os clientes para oferecer esses serviços como parte do relacionamento de cobrança do CSP.

## <a name="new-capabilities-added-to-the-insights-reports-in-partner-center"></a><a name="6"></a>Novas funcionalidades adicionadas aos relatórios de Insights no Partner Center 


### <a name="categories"></a>Categorias

- Data: 16/07/2020
- Capacidades
 
 
### <a name="impacted-audience"></a>Público-alvo afetado

Todas as organizações parceiras na Microsoft Partner Network 

### <a name="summary"></a>Resumo

Novas funcionalidades foram adicionadas aos relatórios do Insights no Partner Center, permitindo que os parceiros realizem análises de dados ad hoc com base nas suas necessidades empresariais e tenham uma visão holística do treinamento adotado pelos funcionários da sua organização.  Essas novas funcionalidades aumentam o conjunto robusto de [relatórios existentes do Insights](../partner-center-insights.md).

Visão geral das novas funcionalidades:

- **Exportação de dados brutos** habilitados para detalhes do cliente, detalhes da assinatura, uso do Microsoft Azure, uso do Office 365, uso do Dynamics 365, uso do EMS (Enterprise Mobility and Security), uso do Power BI, treinamento e certificações e Microsoft Learn. Para obter mais informações, consulte [este artigo](../pci-download-reports.md).

- **Novo painel de treinamento** que fornece uma visão holística do treinamento realizado pelos funcionários na sua organização. Para obter mais informações, consulte [este artigo](../pci-training-dashboard.md).

### <a name="what-is-insights"></a>O que são Insights?

O painel do Insights no Partner Center dá acesso a uma exibição unificada de 360º dos KPIs (indicadores chave de desempenho) da sua organização parceira em produtos de nuvem, como o Office 365, o Azure e o Dynamics 365, e em modelos de licenciamento, como o CSP (Provedor de Soluções na Nuvem) e o EA (Contrato Enterprise). Insights inteligentes são entregues a você para que você entenda claramente seus negócios com a Microsoft e possa tomar decisões orientadas por dados com confiança.   

Para ver esse painel, selecione **Insights** no menu de navegação esquerdo do Partner Center. Esse painel usa os mesmos dados subjacentes que a ferramenta de relatório MyInsights atual, mas oferece visuais mais sofisticados e funcionalidades de relatório mais abrangentes, bem como KPIs que incluem clientes, assinaturas, uso de licenças, competências e benefícios. Verifique-o com frequência, pois continuaremos trazendo novos relatórios e funcionalidades para o painel do Insights.    

### <a name="next-steps"></a>Próximas etapas

Incentivamos você a explorar essas novas funcionalidades e a compartilhar os seus comentários usando as ferramentas de comentários embutidas disponíveis na guia **Insights**.

## <a name="perpetual-software-in-the-cloud-solution-provider-program"></a><a name="5"></a>Software perpétuo no programa Provedor de Soluções na Nuvem 


### <a name="categories"></a>Categorias

- Data: 15/07/2020
- Ofertas/mercados
- Status: Disponível para o conjunto de provedores indiretos qualificados e os revendedores indiretos. Disponibilidade geral em janeiro de 2021.
 
 
### <a name="impacted-audience"></a>Público-alvo afetado

Revendedores Indiretos do CSP e Provedores Indiretos do CSP qualificados 

### <a name="details"></a>Detalhes

Na Microsoft, estamos comprometidos a dar suporte a todas as organizações durante o percurso de transformação digital. Fazemos isso investindo não apenas em tecnologia, mas também examinando o percurso do cliente e do parceiro de ponta a ponta, da aquisição da nossa tecnologia à experiência de suporte. Por meio desse trabalho, identificamos uma oportunidade de transformação, investindo na nossa plataforma de comércio.  Também estamos simplificando os programas de licenciamento enquanto melhoramos a nova experiência de comércio, para que possamos aprimorar como fornecemos mais oportunidades para os nossos parceiros. 

Conforme aprimoramos nossas funcionalidades de comércio para fornecer aos clientes e parceiros uma melhor experiência de compra, estamos expandindo o portfólio de ofertas para incluir a disponibilidade de licenças de software perpétuas no programa CSP (Provedor de Soluções na Nuvem). A adição de licenças de software no CSP permite que os parceiros participantes se concentrem em oferecer soluções diversas e serviços gerenciados aos clientes. Ela também capacita a transição dos clientes para a nuvem e dá aos parceiros a flexibilidade necessária para ambientes de nuvem híbrida.   

Desde 1º de julho, estamos ampliando essa funcionalidade por meio de um conjunto de provedores indiretos qualificados e os revendedores indiretos deles que poderão negociar licenças para software local com clientes no programa CSP. Os parceiros qualificados são gerenciados pela Microsoft e precisam ser provedores indiretos do CSP ativos e Distribuidores Autorizados da Microsoft do programa Open License operando em uma geografia com suporte no momento para a venda de software no CSP. A partir de janeiro de 2021, a Microsoft escalará essa funcionalidade ainda mais e habilitará todos os parceiros de fatura direta, provedores indiretos e revendedores indiretos do CSP para vender licenças de software no programa CSP.    

### <a name="next-steps"></a>Próximas etapas

- Examine todos os materiais de preparação na coleção da galeria de recursos de Preparação de Operações:  [Software perpétuo no programa Provedor de Soluções na Nuvem](https://partner.microsoft.com/resources/collection/software-in-csp#/).
